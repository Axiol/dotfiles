## Critères d'audit (par priorité)

### Bloquant — perceptibilité et navigation

- **Structure sémantique** : landmarks HTML5 corrects (`<main>`, `<nav>`, `<header>`, `<footer>`, `<aside>`) — un seul `<main>` par page
- **Hiérarchie des titres** : `<h1>` unique par page, pas de saut de niveau (`h1 → h3` sans `h2`)
- **Images** : `alt` présent et pertinent (pas vide si image porteuse de sens, vide si décorative)
- **Formulaires** : chaque `<input>`, `<select>`, `<textarea>` a un `<label>` associé (via `htmlFor` ou `aria-labelledby`)
- **Contraste** : ratio minimum 4.5:1 (texte normal), 3:1 (grand texte / UI) — vérifier avec les couleurs Tailwind utilisées
- **Focus visible** : jamais `outline: none` sans alternative visible, `focus-visible` présent

### Important — opérabilité clavier et ARIA

- **Navigation clavier** : tous les éléments interactifs accessibles via Tab/Shift+Tab dans un ordre logique
- **Pièges clavier** : modale, drawer ou menu qui piège le focus sans `Escape` pour s'échapper
- **ARIA roles** : rôles corrects (`role="button"` sur un `<div>` cliquable uniquement si `<button>` impossible), pas de rôles redondants (`<button role="button">`)
- **ARIA états** : `aria-expanded`, `aria-selected`, `aria-checked`, `aria-current` synchronisés avec l'état React
- **ARIA labels** : `aria-label` ou `aria-labelledby` sur les éléments sans texte visible (icônes, boutons icon-only)
- **Liens** : textes descriptifs — jamais "Cliquez ici", "En savoir plus" sans contexte (`aria-label` si besoin)
- **Bouton vs lien** : `<button>` pour les actions, `<a href>` pour la navigation — ne pas inverser
- **Animations** : respecter `prefers-reduced-motion` — désactiver ou réduire les transitions CSS/JS

### Important — compréhensibilité

- **Langue** : `lang="fr"` sur `<html>`, `lang` sur les fragments dans une autre langue
- **Erreurs formulaires** : messages d'erreur associés au champ via `aria-describedby`, pas uniquement par couleur
- **Timeout / mises à jour** : régions live (`aria-live="polite"`) pour les contenus mis à jour dynamiquement (ex : résultats de recherche, alertes)

### Mineur — robustesse

- **IDs uniques** : pas d'`id` dupliqué dans le DOM (problème fréquent en React avec les listes)
- **Tableaux** : `<th>` avec `scope`, `<caption>` si nécessaire
- **Vidéo / audio** : transcription ou sous-titres mentionnés (même si le contenu est géré par Redbee)
- **PDF / documents** : liens vers des ressources alternatives accessibles si documents non accessibles

## Patterns React/OAOS spécifiques

### Gestion du focus sur navigation SPA

Next.js App Router ne restore pas le focus automatiquement entre les navigations. Vérifier que le focus est replacé sur le `<h1>` ou le `<main>` après un changement de route.

### Composants `libs/ui/`

Les composants de `libs/ui/` sont partagés entre `one-site` et `entreprise`. Un fix a11y ici impacte les deux apps — vérifier la cohérence dans les deux contextes.

### Icônes sans texte

Pattern attendu pour les boutons icon-only :

```tsx
<button aria-label={'Fermer la modale'}>
  <IconClose aria-hidden={true} />
</button>
```

### Modale / Dialog

- Focus doit être piégé à l'intérieur (`focus-trap`)
- `role="dialog"` + `aria-modal="true"` + `aria-labelledby` pointant vers le titre
- `Escape` ferme la modale et replace le focus sur l'élément déclencheur

### Tailwind CSS 4 et contraste

Les couleurs Tailwind 4 ont des palettes différentes de Tailwind 3. Vérifier les ratios de contraste avec les utilitaires `text-*` / `bg-*` en contexte réel, pas uniquement en théorie.
